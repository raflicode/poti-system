# UI GUIDELINES

## Design Philosophy

POTI System is a cashier and operations management system used by students.

The interface should prioritize:

- Simplicity
- Clarity
- Fast workflow
- Human-centered design
- Ease of use
- Minimal learning curve

Avoid:
- Futuristic UI
- Cyberpunk style
- Glassmorphism overload
- Complex animations
- Overly decorative elements

# Dashboard

Dashboard should show:

Admin:
- Daily Revenue
- Total Transactions
- Product Count
- Low Stock Products

Piket:
- Today's Schedule
- Today's Transactions
- Quick Cashier Access

---

# Cashier Page

Priority:
Transaction speed

Requirements:

- Fast product search
- Large action buttons
- Clear total amount
- Clear change amount

Important:

Cashier should be usable within a few seconds without training.

---

# Responsive Design

Support:

- Laptop
- Desktop
- Tablet

Mobile support is optional for MVP.

---

# Empty States

Every page must provide:

- Empty state illustration or icon
- Helpful message
- Call to action

Example:

"No products available yet."
"Click Add Product to create your first product."

---

# Notifications

Use toast notifications.

Success:
Green

Warning:
Orange

Error:
Red

Keep messages concise.

---

# Accessibility

- Clear contrast
- Readable font sizes
- Keyboard-friendly forms
- Large click targets

---

# Overall Feeling

The application should feel like:

"A modern campus operational system that is practical, clean, and easy to use."

Not:

"A futuristic startup dashboard."
